# This is TanStack Start (not Next.js)

This site was migrated off Next.js. It now runs on **TanStack Start** (TanStack Router + Vite). TanStack Start is pre-1.0 and moves fast — its APIs may differ from your training data. Before writing framework code, read the version-matched docs bundled in the package:

- `node_modules/@tanstack/react-start/skills/` — Start docs, incl. `lifecycle/migrate-from-nextjs/SKILL.md` (server routes, head, navigation).
- `node_modules/@tanstack/router-core/skills/router-core/` — routing, file naming, SSR, head management.

## Layout

- `src/routes/__root.tsx` — root document (`<html>` shell, `head()` metadata, fonts/CSS links). Uses `shellComponent`, not `component`.
- `src/routes/index.tsx` — the single page (`createFileRoute('/')`).
- `src/routes/og[.]png.tsx` — server route at `/og.png`. Dynamic OG image via **satori** + **@resvg/resvg-js**. Reads the font from disk (`dist/client/fonts` in prod, `public/fonts` in dev) — do NOT self-fetch over HTTP (breaks inside a container).
- `src/styles.css` — Tailwind v4 (`@import "tailwindcss"`) + self-hosted Geist (`@fontsource-variable/geist`). Sets `--font-geist-sans` / `--font-geist-mono`.
- `src/routeTree.gen.ts` — generated by the router plugin. Tracked in git; do not hand-edit.
- `server.mjs` — production entry. srvx serves `dist/client` statically and falls through to the SSR handler.
- `public/` — static assets (`icon.svg`, `favicon.ico`, `fonts/`).

## Build & run

- Dev: `npm run dev` (port 3000).
- Build: `npm run build` → `dist/server/server.js` + `dist/client/`. (No Nitro / `.output` in this version.)
- Prod: `npm start` → `node server.mjs`, port 3000 (honors `PORT`).
- Test: `npm run test` (`node --test tests/*.test.mjs`). Lint: `npm run lint`.

## Deploy (Dokploy on VPS)

Build via the multi-stage `Dockerfile` (node:20-alpine). The runner copies `dist`, `node_modules` (srvx + the native `@resvg/resvg-js` are not bundled), `server.mjs`, `package.json`; runs `node server.mjs`; exposes port 3000. No env vars required (static site). Point Dokploy at the repo + Dockerfile.
