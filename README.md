# luancunha.dev

🔗 **[luancunha.dev](https://luancunha.dev)**

Portfolio site of **Luan Cunha** — senior full stack developer (~8 years). Single landing page that presents technical range, builds trust, and drives qualified WhatsApp conversations. Terminal / Matrix mood, kept professional for business buyers.

Built with [TanStack Start](https://tanstack.com/start) (TanStack Router + Vite), React 19, and Tailwind v4.

## Features

- Server-rendered single landing page with TanStack Start.
- i18n via i18next — `pt-BR` default, English under `/en`.
- Dynamic Open Graph image at `/og.png` (satori + @resvg/resvg-js), localized via `?lang`.
- Self-hosted [Geist](https://vercel.com/font) (sans + mono) through `@fontsource-variable`.
- Privacy-friendly analytics (self-hosted Umami).

## Getting Started

```bash
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

Edit the page at `src/routes/index.tsx`. The root document (`<html>` shell, head metadata, fonts/CSS) lives in `src/routes/__root.tsx`.

## Scripts

| Command | Description |
| --- | --- |
| `npm run dev` | Dev server on port 3000. |
| `npm run build` | Production build → `dist/server/server.js` + `dist/client/`. |
| `npm start` | Run production server (`node server.mjs`, honors `PORT`). |
| `npm test` | Run tests (`node --test tests/*.test.mjs`). |
| `npm run lint` | ESLint. |

## Project Layout

- `src/routes/__root.tsx` — root document and head metadata (uses `shellComponent`).
- `src/routes/index.tsx` — the page (`createFileRoute('/')`).
- `src/routes/og[.]png.tsx` — server route for the dynamic OG image.
- `src/styles.css` — Tailwind v4 + self-hosted Geist.
- `src/routeTree.gen.ts` — generated by the router plugin; tracked in git, do not hand-edit.
- `server.mjs` — production entry (srvx serves `dist/client`, falls through to SSR).
- `public/` — static assets (`icon.svg`, `favicon.ico`, `fonts/`).

See [AGENTS.md](./AGENTS.md) for deeper notes on the framework and conventions.

## Deploy

Deployed with [Dokploy](https://dokploy.com) on a VPS via the multi-stage `Dockerfile` (node:20-alpine). The runner copies `dist`, `node_modules`, `server.mjs`, and `package.json`, then runs `node server.mjs` on port 3000. No env vars required. Point Dokploy at the repo + Dockerfile.
